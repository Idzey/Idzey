## обо мне

Я fullstack-разработчик. Делаю понятные, функциональные и поддерживаемые веб- и мобильные приложения — от интерфейса до backend-логики.

Мне нравится работать с продуктом целиком: разобрать сценарий, спроектировать API, подключить его к интерфейсу, добавить авторизацию, хранение данных, real-time — и довести до рабочего состояния.

Параллельно активно участвую в хакатонах. Это формат, где нужно за ограниченное время понять задачу, договориться внутри команды, распилить роли и собрать рабочий MVP. Из таких заездов вытаскиваю реальный опыт по архитектуре, командной работе и быстрым техническим решениям.

Мои основные интересы:

- fullstack-приложения и SaaS-продукты
- интерфейсы на react / vue
- frontend-архитектура (FSD, чистая структура, переиспользование)
- backend на node.js (express, fastify)
- REST API, авторизация, безопасность
- real-time (websockets, SSE, STOMP)
- мобильная разработка через React Native / Expo / Capacitor
- разработка MVP на хакатонах
- код, который легко читать, поддерживать и расширять

## проекты

### [TeamPilot](https://github.com/42team-ru/teampilot) — AI-ассистент проектного менеджера

Бот в Telegram, который сам разбирает чат и встречи команды, извлекает задачи и ведёт канбан в YouGile.

**моя роль:** Chrome-расширение для real-time анализа онлайн-встреч.

**стек расширения:** `WXT` `React` `TypeScript` `Tailwind` `Manifest V3` `@stomp/stompjs` `chrome.storage`

Что сделал:

- захват аудио активной вкладки браузера (offscreen + MediaRecorder, WebM-чанки)
- стриминг чанков по 30 секунд через STOMP/WebSocket на backend
- авторизация через Telegram pairing-code + JWT polling
- sidepanel с живым логом транскрипции, найденными задачами и summary встречи
- live-подсказки во время звонка («похожая задача уже есть»)
- переключение тем (system / light / dark), хронологический лог с автоскроллом
- service worker, оркестрирующий состояние встречи и соединение

> Хакатонный проект "Цифровой прорыв".

### [NaturalQL](https://github.com/42team-ru/mpit-final) — natural language → SQL

Корпоративная платформа для аналитики. Пользователь задаёт вопрос на русском — система возвращает SQL, результат и график. Финал хакатона МПИТ 2025.

**моя роль:** frontend.

**стек:** `React 19` `React Router 7` `TypeScript` `Tailwind CSS 4` `Recharts` `XYFlow` `Zustand` `Vite` `SSE`

Что внутри фронта:

- чат-интерфейс с потоковой генерацией SQL через Server-Sent Events
- визуализация результатов запроса: таблицы и графики (Recharts)
- семантический слой через визуальный редактор схем (XYFlow)
- роли в интерфейсе: аналитик, менеджер, офицер безопасности
- аудит-журнал и работа с ошибками валидации запросов

### [fit-tracker](https://github.com/Idzey/fit-tracker) — SaaS-платформа для фитнес-тренеров

Mono-репозиторий с backend и mobile-приложением. Тренеры ведут клиентов, шаблоны тренировок, подписки; клиенты выполняют тренировки в приложении. Полностью мой проект — от схемы БД до интерфейса.

**стек:**

- **backend:** `Fastify` `Prisma` `PostgreSQL` `S3` `SSE` `Expo Push` `Pino` `Sentry`
- **mobile:** `Expo Router` `React Native` `NativeWind` `TanStack Query` `Zustand`

Что реализовал:

- авторизация ролей (тренер / клиент), подписки и доступ к функционалу
- шаблоны тренировок, клиенты, расписание
- мобильное приложение со счётчиками подходов и завершением тренировки
- **офлайн-режим:** персистентность TanStack Query + очередь мутаций для повтора при возврате сети
- push-уведомления через Expo Push
- SSE для realtime-обновлений
- мониторинг: Pino-логи, Sentry, health checks (`/health/live`, `/health/ready`)
- запуск в Docker (`docker-compose` для PostgreSQL и MinIO)
